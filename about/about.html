About

Many people may already be familiar with the term "tesseract" from its use in the Marvel Avengers comics and movies, where it was the name given to the "Infinity Stone" controlling space in their universe.

While this was an appropriate artistic application, the term tesseract itself actually originates in the study of higher-dimensional geometry. Visualize a flat, 2-D square, being somehow 'expanded' or stretched into a 3-D cube. Now try and imagine the same sort of process being done to the cube, where it is 'expanded' into a 4-Dimensional object. While we can't truly visualize this object, mathematicians can work with it in an abstract way, and they named this 4-D analog the tesseract, AKA the hypercube.

Using an Object Oriented Programming Approach was vital to the development of this game. This project as a whole includes over 1600 lines of JavaScript Code, and would have quickly become unmanageable without some organizing principles. I chose to use three main JavaScript Classes to represent the primary Objects involved:

  • A Frame Class: For representing the 3-Dimensional Grid in which the game takes place.
  • A Block Class: For representing and rendering individual 6-sided blocks inside the Frame.
  • A Shape Class: For representing the connected collections of blocks that are controlled as a single entity by the player as the game progresses.

The Frame is constructed dynamically using JavaScript, and is composed of three sets of parallel planes, each rotated into position using css Transforms. These planes are all set to "display: grid;" and the faces of each block are child Elements of these grids.

Each block is rendered as a collection of 3 pairs of divs, for a total of 6 faces. Each parallel pair is placed within the frame by appending the divs to neighboring planes. For instance, a block at the bottom of the frame would have a bottom face in the lowermost plane, and a top face in the grid-plane one level above.

Blocks are then moved inside the frame work by removing and appending each of these 6 divs to different grid areas in the planes making up the frame.

Each Shape is then simply represented by an array of blocks. Because this is the unit which players manipulate as a whole, it made sense to define methods of this class for translating and rotating shapes as a whole. Translating a shape is fairly simple, as it only involves moving each block of a given shape 1 unit in 1 direction.

Rotation was much more complicated, and implementing it required some basic matrix algebra and calculating each blocks position relative to a "keystone" block (outlined in gold) that would serve as the center around which all the other blocks in a shape would rotate.
